# 付費推播受眾包規劃

日期：2026-05-20

## 核心原則

付費推播不要直接賣「全名單」。每一包都要有明確條件、排除規則、預覽人數、樣本名單、發送節奏與成效報表。

高風險名單不建議作為廣告受眾包販售。它應該是客服關懷名單，或作為付費推播的排除條件，避免把正在抱怨、重複點擊或未解決的人再打擾一次。

## 建議受眾包

| 受眾包 | 條件 | 適合用途 | 發送建議 |
| --- | --- | --- | --- |
| 全站活躍包 | 30 天內互動、排除高風險 | 品牌曝光、一般活動 | 每批 100 人、間隔 60 秒 |
| 近 7 天熱名單包 | 7 天內互動、排除高風險 | 限時活動、住房提醒 | 每批 80-100 人、間隔 60 秒 |
| 高點擊互動包 | 曾點擊 postback 或活動按鈕、排除高風險 | 高轉換促銷、再行銷 | 每批 50-80 人、間隔 60-90 秒 |
| 旅宿意圖包 | 摘要、標籤或訊息含房型、日期、價格、地點等線索 | 精準住房導流 | 每批 50 人、間隔 90 秒 |
| 地區包 | inferred_area、location_address、summary 或 tags 符合地區 | 區域店家、在地活動 | 每批 50-100 人、間隔 60 秒 |
| 性別線索包 | inferred_gender 為男性或女性線索 | 需要性別偏好的活動 | 每批 50-100 人、間隔 60 秒 |
| 指定名單包 | 業主提供 LINE userId 或後台人工挑選 | 客製活動、VIP 通知 | 依名單大小分批 |
| 沉睡喚回包 | 30 天以上未互動，且非高風險 | 回流優惠、再喚醒 | 每批 30-50 人、間隔 120 秒 |

## 每包必備欄位

| 欄位 | 說明 |
| --- | --- |
| package_id | 受眾包代碼，例如 active_7d、intent_room、area_tainan |
| package_name | 給業務或客戶看的名稱 |
| filters_json | 實際推播篩選條件 |
| excludes_json | 排除條件，預設排除 high risk |
| estimated_count | 建包當下預覽人數 |
| snapshot_at | 預覽快照時間 |
| sample_users | 樣本 5-20 筆，供人工確認名單品質 |
| price_mode | 固定價、CPM、或專案價 |
| unit_price | 每千人價格或固定價格 |
| send_window | 可發送時段，例如 10:00-21:00 |
| batch_size | 每批人數 |
| interval_seconds | 批次間隔秒數 |
| frequency_cap_days | 同一用戶幾天內不能重複收到付費推播 |

## 建議價格邏輯

| 類型 | 建議計價 |
| --- | --- |
| 全站活躍包 | CPM 較低，適合大量曝光 |
| 近 7 天熱名單包 | CPM 中高，因為互動溫度高 |
| 高點擊互動包 | CPM 高，或加收轉換導向服務費 |
| 旅宿意圖包 | CPM 高，名單少但精準 |
| 地區/性別線索包 | 依條件精細度加價 |
| 指定名單包 | 收設定費加發送費 |

## 操作流程

1. 選受眾包或設定條件。
2. 按預覽，確認人數與樣本。
3. 系統建立名單快照，不用即時全庫重算。
4. 審核推播文案，確認沒有違規、過度承諾或打擾高風險客訴。
5. 建立分時推播，依批次與間隔送出。
6. 匯出報表：總人數、成功、失敗、點擊、尖峰時段、Top clickers。

## 目前系統可先支援的版本

第一階段直接用現有 `/broadcast` 條件：

- 全站活躍包：mode=active30，risk=low 或排除 high。
- 近 7 天熱名單包：mode=active7，risk=low。
- 地區包：mode=area，輸入地區。
- 性別線索包：mode=gender。
- 旅宿意圖包：mode=keyword 或 tag。
- 指定名單包：mode=custom。

第二階段再補：

- 點擊互動包：用 postback 統計建立 click_count 條件。
- 沉睡喚回包：新增 inactive_days 條件。
- 受眾包保存：把 filters_json、excludes_json、price_mode、snapshot_at 存成可重用設定。
- 頻率上限：同一 userId 在 N 天內不得收到第二次付費推播。
